## 权限错误

### 🔧 `parse/handle error: Error: EPERM: operation not permitted`

**症状**：OpenClaw 无法写入临时文件，报权限错误

**解决方案**：
```bash
sudo chown -R `whoami`:staff ~/.openclaw && chmod -R u+rw ~/.openclaw
```
修复权限后重启网关即可。

## 启动问题

### 📡 启动命令选择

- `openclaw gateway restart`：重启 launchd 服务，适合服务模式
- `openclaw gateway stop && sleep 2 && openclaw gateway --force`：强制杀掉前台进程，干净重启，适合前台终端运行
- **推荐**：当前台终端运行网关时，使用第二种方式避免端口冲突

## 配置问题

### 🧭 配置向导显示 `网络错误，无法验证 API Key`

**原因**：OpenClaw 配置向导内部验证 API Key 时遇到连接问题

**解决方案**：
- 这是 OpenClaw 本身的连接 bug，不影响已经手动配置好的网关
- 网关已经成功启动，可以直接访问网页端使用
- 如果仍然超时，使用 `./chat.sh` 脚本绕过即可

### 🔍 为什么联网搜索需要单独配置 API？

**架构理解**：
- ✅ **大模型**：提供文本理解和生成能力，本身不联网
- ✅ **OpenClaw 工具**：提供联网搜索、文件读写等外部能力
- ✅ **需要单独配置 API**：搜索服务是第三方提供的，需要你的 API Key 才能调用

**结论**：如果不需要搜索最新信息，只需要聊天、写代码，不用配置搜索 API 也能正常使用。

## 使用问题

### ❓ 为什么总是答非所问？

**常见原因**：
1. 上下文太长，模型理解混乱
2. OpenClaw 默认启用了很多工具，错误调用工具导致答非所问
3. 之前多次超时失败导致会话状态异常

**解决方案**：
- 新建会话：网页端左侧边栏会话列表点击 "+ 新建"
- 提问时明确说明：`请直接回答，不要调用工具`
- 如果还是不行，刷新网页重新进入就是新会话

### 🌐 网页端 v2026.3.13 找不到新建会话按钮

**解决方案**：
1. 左侧边栏展开"会话"，会话列表顶部有 "+ 新建"
2. 或者直接刷新网页，重新打开控制面板就是新会话
3. 在聊天输入框输入 `/new` 回车也能新建会话

## 卸载问题

### 🗑️ 如何卸载 Ark Helper？

**完整卸载步骤**：
```bash
rm -rf "$HOME/.ark-helper"
rm /opt/homebrew/bin/ark-helper
```

**验证**：执行 `which ark-helper` 没有输出说明卸载成功。
