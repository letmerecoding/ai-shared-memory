# 提示词优化任务规则（2026-03-11 生效）

## 迭代管理规则
1. 历史保留：迭代提示词时必须完整保留所有历史版本的提示词
2. 版本管理：每次迭代必须创建唯一版本号，版本号格式建议：vX.Y.Z（如v1.0.0）
3. 版本说明要求：每次迭代必须编写完整的版本说明，包含以下内容：
   (1) 此次迭代发现了哪些问题
   (2) 此次迭代解决了哪些问题
   (3) 此次迭代所发现的问题中有哪些需要人为介入
   (4) 给出此次迭代所发现的需要人为介入的问题的具体解决方案

## 存储规则
1. 隔离存储：每个关联任务的提示词单独保存，互不干扰
2. 存储结构建议：
   ```
   prompt-optimization/
     ├── task-1-新闻日报任务/
     │   ├── versions/
     │   │   ├── v1.0.0.md
     │   │   ├── v1.1.0.md
     │   │   └── ...
     │   └── current.md
     ├── task-2-统计指标收集任务/
     │   └── ...
     └── ...
   ```

## 邮件通知规则
1. 发送周期：每周迭代任务完成后发送
2. 发送方式：每个提示词的迭代信息单独发送一封邮件，不汇总
3. 邮件内容必须包含：
   (1) 完整的版本说明
   (2) 迭代前的当前版本提示词
   (3) 迭代后的新版本提示词
   (4) 两个版本之间的文本差异对比
