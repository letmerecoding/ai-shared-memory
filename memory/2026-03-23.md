# 2026-03-23 对话记录

## 对话记忆管理方案讨论

### 用户提出的想法
用户每天需要和多个大模型对话，希望建立一套自动记忆管理机制：
1. 每个大模型将对话以文档形式按日期归档存储
2. 让另一个大模型阅读前一天/当天的归档文档
3. 汇总对话信息，将有价值或需要长期记忆的信息提炼到记忆仓库
4. 让记忆框架随着聊天不断丰富完善

### 方案确认
现有的OpenClaw记忆框架已经支持基础结构：
- **每日归档**：`memory/YYYY-MM-DD.md` 按日期存放每天对话
- **长期仓库**：`MEMORY.md` 存放提炼后的长期价值信息
- **定期维护**：心跳检查时定期更新长期记忆

可以在现有基础上，增加更自动化的每日汇总流程，通过定时任务自动完成每日提炼。

### 后续计划
配置cron定时任务，每天固定时间自动执行：
1. 读取当日/前一日的对话归档文件
2. 提炼有长期保留价值的信息
3. 更新整合到`MEMORY.md`长期记忆仓库
