# 定时任务默认规则（永久生效）
## 1. 模型分配规则
- 未指定模型：1个生成模型 + 1个测试模型，不同日期轮流使用，优先调用速度快的模型
- 指定生成模型：剩余所有模型自动作为测试模型，按优先级轮流分配
- 模型优先级顺序：deepseek-v3.2 > ark-code-latest > doubao-seed-code > doubao-seed-2.0-code > glm-4.7 > kimi-k2.5

## 2. 指标统计规则
- 默认行为：所有任务都统计生成模型和测试模型的各项指标，除非明确说明"不要统计指标"
- 未指定指标：根据任务场景生成建议指标列表，请求用户确认
- 明确"自己选指标"：自动选择合适的指标，无需确认

## 3. 通用规则
- 所有定时任务都默认加入自动降级机制：当前模型失败自动调用下一个优先级模型，保证任务100%成功
- 所有统计数据永久保存，定期（每周）分析优化
